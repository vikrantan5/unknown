## Important Directories Explained

### 🎯 `/futurproctor/` - Your Working Directory
This is where you'll spend most of your time. All commands should be run from here.

**Key Files:**
- `manage.py` - Your main tool for all Django operations
- `.env` - Configuration file (you need to create this)
- `venv/` - Virtual environment (you need to create this)

### 🔧 `/futurproctor/futurproctor/` - Project Configuration
Django project configuration folder (same name can be confusing but standard).

**Key Files:**
- `settings.py` - Database, installed apps, middleware, static files, etc.
- `urls.py` - Maps URLs to views (main routing)

### 🚀 `/futurproctor/proctoring/` - Main Application
This is where the actual application logic lives.

**Key Directories:**
- `static/` - CSS, JavaScript, images for the frontend
- `templates/` - HTML files for the user interface
- `ml_models/` - AI/ML model code and configurations
- `migrations/` - Database schema changes (auto-generated)

**Key Files:**
- `models.py` - Database structure (tables, fields, relationships)
- `views.py` - Main application logic and request handling
- `admin_views.py` - Admin panel specific functionality
- `student_exam_views.py` - Student exam interface logic
- `urls.py` - Application-specific URL routing
- `groq_service.py` - AI service integration
- `email_service.py` - Email notification system

### 📦 AI/ML Files
- `yolo11s.pt` (19MB) - Pre-trained YOLO model for object detection
- `dlib-19.22.99-cp310-cp310-win_amd64.whl` - Face detection library
- `ml_models/` - Custom model implementations

### 📓 Research & Documentation
- `Object_detection_model_training_rcnn.ipynb` - R-CNN model training
- `Yolov8_Object_detection.ipynb` - YOLO model training
- `Research paper.pdf` - Academic documentation

## Typical Workflow

### 1. Initial Setup
```
1. Clone repository → ProctoringME/
2. cd ProctoringME/futurproctor/
3. Create venv → venv/
4. Create .env file
5. Install dependencies
6. Setup database
7. Run migrations
```

### 2. Daily Development
```
1. cd ProctoringME/futurproctor/
2. Activate venv
3. python manage.py runserver
4. Make changes in proctoring/
5. Test in browser
6. Ctrl+C to stop server
```

### 3. Adding Features
```
1. Edit models.py → Database changes
2. python manage.py makemigrations → Create migration
3. python manage.py migrate → Apply to DB
4. Edit views.py → Add logic
5. Edit templates/ → Add/modify HTML
6. Edit urls.py → Add routes
7. Test the feature
```

## Dependencies Location

### Python Packages
After `pip install -r requirements.txt`, packages are installed in:
```
venv/lib/python3.10/site-packages/
```

### Static Files
- **Development:** Served from `proctoring/static/`
- **Production:** Collected to `staticfiles/` (after collectstatic)

### Media Files
- User uploads go to: `media/`
- Configured in: `settings.py` (MEDIA_ROOT, MEDIA_URL)

## .gitignore Highlights

These files/folders are ignored by git (don't get committed):
```
venv/                # Virtual environment
__pycache__/         # Python cache
*.pyc                # Compiled Python
.env                 # Environment secrets
media/               # User uploads
staticfiles/         # Collected static files
db.sqlite3           # SQLite database (if used)
```
